# 决策记录

## 2026-06-21：独立于聆听训练项目

- 决策：单词记忆 App 使用独立的 `project-001-个人雅思单词记忆` 目录。
- 备选：直接覆盖 Workspace 根目录“聆听训练”。
- 理由：根目录存在未提交用户改动，覆盖会造成数据丢失和项目混淆。
- 影响：单词记忆拥有独立源码、构建流程、Bundle ID 和交付物。

## 2026-06-21：采用 PWA + WKWebView + AltStore

- 决策：核心功能以离线 PWA 实现，使用轻量 WKWebView 容器构建 IPA。
- 备选：纯 SwiftUI；只做 Safari PWA。
- 理由：用户无 Mac，现有 Windows + GitHub Actions + AltStore 链路已经验证可行。
- 影响：业务逻辑可在 Windows 测试，iOS 容器保持最小化。

## 2026-06-21：主屏幕名称和 Bundle ID

- 决策：显示名为“单词TUO”，Bundle ID 为 `com.personal.wordtuo`。
- 理由：满足用户最新命名要求，并避免覆盖 `com.personal.listeningtraining`。

## 2026-06-21：本地优先数据

- 决策：第一版使用 localStorage 保存词库、复习记录和设置，支持 JSON 导出恢复。
- 理由：个人单设备优先，减少服务器依赖；后续可迁移到 IndexedDB/iCloud。

## 2026-06-21：v2 采用主动回忆 + SM-2

- 决策：新词经过认识、主动回忆和例句填空三阶段；复习间隔由 SM-2 的重复次数、难度因子和质量评分计算。
- 备选：继续使用固定 1/3/7/15/30 天等级表。
- 理由：SM-2 能根据个人反馈动态调整间隔，同时保持三档操作适合碎片时间。
- 影响：本地数据和备份升级为版本 2，读取时兼容 v1。

## 2026-06-21：21天混合抽查

- 决策：每21天生成最多30词的选择、拼写、例句填空混合卷，80分及格。
- 理由：兼顾词义识别、主动输出和上下文应用；错词回到当天复习队列。
- 影响：新增可续做试卷、成绩历史和错词重测数据。

## 2026-06-21：影视语境仅接合法免费来源

- 决策：提供统一片段来源接口和 IndexedDB 缓存；PlayPhrase.me 没有确认免费公开接口时只提供外部搜索，不抓取片段。
- 备选：抓取网站视频；接入付费接口。
- 理由：用户要求完全免费，同时必须避免版权和接口滥用风险。
- 影响：当前实验模块能快速返回3个语境入口，但不承诺精准的5-20秒影视片段。

## 2026-06-22：五种记忆法与可播放片段

- 决策：系统学习同时提供词根词缀、联想钩子、词族联结、音节拼写和语境记忆；影视语境采用外部搜索、保存公开视频链接、App内播放的流程。
- 备选：只显示词根；自动抓取 PlayPhrase.me；只保留外部链接。
- 理由：多通道记忆更适合不同单词；在无免费授权接口的前提下，保存合法公开视频是可实际播放且可维护的方案。
- 影响：数据升级为 v3，每词可保存自定义记忆材料和最多3个5–20秒片段。

## 2026-06-22：第三学习阶段改为影视语境

- 决策：新词第三阶段不再使用普通例句填空，改为播放已保存片段并根据字幕挖空回忆目标词。
- 无片段处理：提供“查找并添加片段”和“本词先跳过影视”，不阻断当日学习。
- 保留范围：三周抽查仍保留例句填空题，用于阶段性能力检查，而非初次记忆。
- 理由：影视场景负责建立记忆锚点，例句填空更适合作为后续测试。
