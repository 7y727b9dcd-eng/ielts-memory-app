# 决策记录

## 2026-06-21：独立于聆听训练项目

- 决策：单词记忆 App 使用独立的 `project-001-个人雅思单词记忆` 目录。
- 备选：直接覆盖 Workspace 根目录“聆听训练”。
- 理由：根目录存在未提交用户改动，覆盖会造成数据丢失和项目混淆。
- 影响：单词记忆拥有独立源码、构建流程、Bundle ID 和交付物。

## 2026-06-21：采用 PWA + WKWebView + AltStore

- 决策：核心功能以离线 PWA 实现，使用轻量 WKWebView 容器构建 IPA。
- 备选：纯 SwiftUI；只做 Safari PWA。
- 理由：用户无 Mac，现有 Windows + GitHub Actions + AltStore 链路已经验证可行。
- 影响：业务逻辑可在 Windows 测试，iOS 容器保持最小化。

## 2026-06-21：主屏幕名称和 Bundle ID

- 决策：显示名为“单词TUO”，Bundle ID 为 `com.personal.wordtuo`。
- 理由：满足用户最新命名要求，并避免覆盖 `com.personal.listeningtraining`。

## 2026-06-21：本地优先数据

- 决策：第一版使用 localStorage 保存词库、复习记录和设置，支持 JSON 导出恢复。
- 理由：个人单设备优先，减少服务器依赖；后续可迁移到 IndexedDB/iCloud。

## 2026-06-21：v2 采用主动回忆 + SM-2

- 决策：新词经过认识、主动回忆和例句填空三阶段；复习间隔由 SM-2 的重复次数、难度因子和质量评分计算。
- 备选：继续使用固定 1/3/7/15/30 天等级表。
- 理由：SM-2 能根据个人反馈动态调整间隔，同时保持三档操作适合碎片时间。
- 影响：本地数据和备份升级为版本 2，读取时兼容 v1。

## 2026-06-21：21天混合抽查

- 决策：每21天生成最多30词的选择、拼写、例句填空混合卷，80分及格。
- 理由：兼顾词义识别、主动输出和上下文应用；错词回到当天复习队列。
- 影响：新增可续做试卷、成绩历史和错词重测数据。

## 2026-06-21：影视语境仅接合法免费来源

- 决策：提供统一片段来源接口和 IndexedDB 缓存；PlayPhrase.me 没有确认免费公开接口时只提供外部搜索，不抓取片段。
- 备选：抓取网站视频；接入付费接口。
- 理由：用户要求完全免费，同时必须避免版权和接口滥用风险。
- 影响：当前实验模块能快速返回3个语境入口，但不承诺精准的5-20秒影视片段。

## 2026-06-22：五种记忆法与可播放片段

- 决策：系统学习同时提供词根词缀、联想钩子、词族联结、音节拼写和语境记忆；影视语境采用外部搜索、保存公开视频链接、App内播放的流程。
- 备选：只显示词根；自动抓取 PlayPhrase.me；只保留外部链接。
- 理由：多通道记忆更适合不同单词；在无免费授权接口的前提下，保存合法公开视频是可实际播放且可维护的方案。
- 影响：数据升级为 v3，每词可保存自定义记忆材料和最多3个5–20秒片段。

## 2026-06-22：第三学习阶段改为影视语境

- 决策：新词第三阶段不再使用普通例句填空，改为播放已保存片段并根据字幕挖空回忆目标词。
- 无片段处理：提供“查找并添加片段”和“本词先跳过影视”，不阻断当日学习。
- 保留范围：三周抽查仍保留例句填空题，用于阶段性能力检查，而非初次记忆。
- 理由：影视场景负责建立记忆锚点，例句填空更适合作为后续测试。

## 2026-07-03：增加高中人教词库并支持词库切换

- 决策：内置“高中人教版”词库，与“雅思词库”并列；学习、复习、统计和词库页按当前学习词库过滤。
- 当前范围：导入新人教版高中英语必修第一册至第三册、选择性必修第一册至第四册，共 2525 条词汇/短语。
- 备选：只导入必修三册；只保留雅思词库并靠标签筛选。
- 理由：用户当前要求是能自由选择雅思和高中三年词汇；2019 新人教版高中英语完整学习路径通常包含 3 册必修 + 4 册选择性必修。
- 影响：本地数据升级到 v4，单词新增 `catalog` / `catalogs` 字段；高中词库以 JSON 和 JS 两种形式随 PWA 一起缓存，离线也可导入。
- 后续复核：若学校使用的教材版本或老师指定词表不同，可替换为同格式 JSON。

## 2026-07-03：安卓 APK 采用 WebView 容器

- 决策：新增 Android 原生 WebView 容器，直接加载现有 `04-实现/web` 离线资源；GitHub Actions 生成可安装的 debug 签名 APK。
- 显示名：沿用“单词TUO”。
- 包名：`com.personal.wordtuo.android`。
- 备选：重写原生 Android；只让用户用浏览器打开 PWA；使用第三方在线打包服务。
- 理由：当前核心功能已在 Web/PWA 中实现，WebView 容器能最快生成可下载安装的 APK，同时保持安卓和 iPhone 业务逻辑一致。
- 影响：安卓 APK 和 iOS IPA 使用同一套网页学习功能；安卓安装时需要用户允许“安装未知来源应用”。
- 后续复核：若未来要上架应用商店，需要改为 release 正式签名、隐私政策和应用商店合规材料。

## 2026-07-03：v12 优化安卓流畅度和词汇内容

- 决策：底部导航切换改为按当前页面懒渲染；词库页一次只渲染 40 个词，用户点击“加载更多”继续加载。
- 决策：词库页默认跟随当前学习词库；学习、复习、三周抽查和统计都按当前词库过滤。
- 决策：音标和发音采用在线词典懒加载，优先读取 Free Dictionary API 的音标和音频；缺音频时使用在线发音地址兜底；仍保留系统朗读兜底。
- 决策：词根记忆从简单占位改为常见词根、前缀、后缀组合生成，并对旧的自动模板进行迁移刷新。
- 理由：安卓 WebView 对一次性渲染 2500 多条词库较敏感；音频和音标不适合一次性批量抓取，应在学习或朗读当前词时按需补充并本地缓存。
- 影响：本地数据升级到 v5，新增 `audioUrl` 和 `lexiconFetchedAt` 字段；旧 v4 数据会自动读取迁移。
- 风险：在线发音依赖网络和第三方词典可用性，失败时退回系统朗读。

## 2026-07-03：v13 朗读按钮、回音修复与记忆法扩展

- 决策：学习卡片和词库列表都把“听”按钮放到英文单词标题旁边，保留单手点击尺寸。
- 决策：朗读统一通过单一播放管理器执行；每次播放前停止旧网络音频、取消系统朗读和未触发的自动朗读定时器。
- 决策：学习页获取在线音标/音频时不再触发整页重渲染，避免自动朗读被重复调度。
- 决策：继续使用 SM-2 遗忘曲线逻辑，依据“不认识/模糊/认识”更新 `easeFactor`、`intervalDays` 和 `nextReview`，复习队列只取 `nextReview <= today` 的单词。
- 决策：记忆方式从五种扩展为八种，新增音标拆读、句子联想、主动回忆、搭配语境和中英反向回忆。
- 理由：用户反馈发音有回音且记忆功能太少；回音根因不是音频文件本身，而是联网补充词典信息后触发页面刷新并再次自动朗读。
- 影响：PWA 缓存升级到 v13；安卓和网页共享同一套修复。

## 2026-07-03：v14 点击反馈降噪

- 决策：将移动端默认蓝色点击高亮替换为低对比墨绿色 `tap-quiet`，按钮、链接、底部导航、折叠项统一使用轻微暗化按压态。
- 决策：保留键盘 `focus-visible` 轮廓，颜色改为界面内的深绿透明色，避免牺牲可访问性。
- 理由：安卓 WebView / 移动浏览器默认蓝色闪烁与当前精致学习型界面不一致，点击时显得突兀。
- 影响：PWA 缓存升级到 v14；仅影响交互视觉反馈，不改变学习数据和业务逻辑。
