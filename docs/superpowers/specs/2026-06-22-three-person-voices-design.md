# 聆听训练：三位人物语音与计划补完设计

日期：2026-06-22

## 目标

把现有按“自然交流、清晰汇报、沉稳说明”区分的语音选项，改成三位稳定、可识别的具体说话人，并修复当前三项实际都降级为同一个设备朗读音色的问题。同时补齐 v1.2 原计划中与真实音源记录、云端可用性和验收有关的缺口。

## 三位人物

| 人物 ID | 界面姓名 | Azure 音色 | SSML 设置 |
|---|---|---|---|
| `lin_xiao` | 林晓 | `zh-CN-XiaoxiaoNeural` | `chat`，强度 `0.55` |
| `chen_yu` | 陈屿 | `zh-CN-YunxiNeural` | `chat`，强度 `0.45` |
| `su_ning` | 苏宁 | `zh-CN-XiaoyiNeural` | 不使用情绪风格，保持默认神经语音 |

界面使用姓名、简洁人物头像和“女声/男声”辅助标签。语气描述只用于试听说明，不再作为主要选项名称。三位人物使用同一段试听文本，便于直接比较。

## 可用性与降级

应用启动及进入训练准备页时检查语音代理：

- 代理健康且三位人物可用：启用三张人物卡片，记住上次选择。
- 未配置代理、健康检查失败或请求超时：三张人物卡片显示“云语音未连接”并禁用，只提供一个“设备语音”选项。
- 训练过程中单次云请求失败：自动切换设备语音，并在播放区域明确显示“本题使用设备语音”。
- 不再让三个不同人物选项播放同一个设备音色。

健康接口在 Azure Secret 缺失时返回不可用状态，不能只报告 Worker 本身在线。

## 接口与缓存

Worker 接口保持白名单模式：

- `GET /v1/health`
- `GET /v1/tts/{scenarioId}?profile=lin_xiao|chen_yu|su_ning&v=2`

语音缓存升级为版本 2，缓存键只包含场景 ID、人物 ID 和语音版本。旧版 `v=1` 音频不复用，避免继续播放旧的机械风格。

Worker 的 SSML 生成支持“有风格”和“无风格”两种人物配置，不接受网页传入任意文本、音色名或 SSML。

## 数据模型与进步比较

`Attempt` 新增或规范化：

- `speakerId`：`lin_xiao`、`chen_yu`、`su_ning` 或 `system`。
- `audioProvider`：`azure` 或 `system`，代表真实声音来源。
- `audioDelivery`：`network`、`cache` 或 `system`，代表音频取得方式。
- `voiceVersion`：当前为 `2`。

`baselineSummary` 同样保存 `speakerId`、`audioProvider` 和 `voiceVersion`。主要进步结论只比较相同人物、相同真实音源提供方和相同语音版本的数据；缓存与联网获取的同一 Azure 音频可合并比较。

旧数据迁移：

- `natural` → `lin_xiao`
- `briefing` → `chen_yu`
- `calm` → `su_ning`
- 旧记录的 `audioSource=system` → `audioProvider=system`
- 旧记录的 `audioSource=azure|cache` → `audioProvider=azure`

无法判断真实音源的旧记录不进入新版主要进步结论，但仍保留在历史记录和导出数据中。

## 界面调整

- 训练准备页：三张人物卡片显示姓名、头像、男女声标签、试听按钮和可用状态。
- 设置页：“默认工作语音”改为“默认说话人”。代理不可用时显示明确诊断和部署说明。
- 播放区：始终显示当前人物姓名及真实音源；降级时显示“设备语音”。
- 周报：显示用于比较的基线人物和音源；因人物或音源不一致而排除的数据单独计数说明。

## 原计划补完范围

本轮代码实施包含：

1. 三位人物 UI、人物音色映射和公平试听。
2. Worker v2 接口、Azure 配置健康检查和缓存版本升级。
3. 真实音源字段、旧数据迁移及进步比较过滤修正。
4. 无代理时单一设备语音降级，不伪装成三个人。
5. 自动化测试、网页版完整流程测试和 iOS 打包资源检查。

以下事项必须依赖外部账号或设备，代码完成不能代替真实验收：

1. 创建 Azure Speech 与 Cloudflare 账号并部署 Worker。
2. 在 Wi‑Fi 和蜂窝网络分别验证 Worker；若 `workers.dev` 均不可达，再部署 Azure Functions East Asia 备用接口。
3. 三位人物实声试听验收。
4. 生成新版 IPA，在 iPhone Safari、主屏 PWA、WKWebView/AltStore 中验证。
5. 四周训练效果验收。

## 测试与验收

- 三位人物向 Worker 发送不同人物 ID，并生成不同 Azure voice 配置。
- 三位人物使用同一试听文本；缓存重听不重新请求 Azure。
- 无代理或健康检查失败时只显示一个设备语音选项。
- 单题云语音失败后训练不中断，作答记录标记真实设备音源。
- Worker 拒绝旧人物 ID、未知人物、未知场景、任意文本和错误版本。
- 进步报告不会混合不同人物、设备语音与 Azure 语音或不同语音版本。
- 旧 IndexedDB 数据仍可读取、导出和重置。
- PWA 离线启动、缓存升级和 iOS Web 资源复制配置通过检查。
