# 全局复利与踩坑日志

> 用于沉淀所有项目中犯过的错误、踩过的坑、走过的弯路，以及未来可以持续复用的方法。目标是避免重复犯错，让经验能够跨项目复用。

## 1. 使用规则

1. 每个项目开始规划前，先按关键词检索本日志中是否存在相关经验。
2. 出现错误、返工、错误假设、工具限制、兼容性问题或流程浪费时，必须新增记录。
3. 形成稳定有效的模板、检查表、命令、方案或决策方法时，必须新增“可复用方法”。
4. 记录必须包含事实、原因、解决办法和预防措施，不能只写结论。
5. 同类问题重复发生时，应链接原记录，并升级预防措施。
6. 项目结束时，将项目 `08-复盘` 中具有跨项目价值的内容同步到本日志。
7. 记录只追加或修订状态，不直接删除；错误信息需要更正时保留变更说明。

## 2. 分类标签

- `需求`：目标、范围、验收理解错误。
- `规划`：拆解、优先级、依赖或时间安排问题。
- `目录`：文件位置、命名、归档或版本混乱。
- `实现`：代码、配置、依赖、兼容性问题。
- `测试`：测试遗漏、环境差异、验收失败。
- `交付`：格式、构建、发布、说明缺失。
- `沟通`：必要信息未确认、错误假设、反馈遗漏。
- `工具`：Codex、插件、命令或外部工具限制。
- `方法`：可跨项目复用的流程、模板或技巧。

## 3. 严重程度

- `P0`：造成数据丢失、安全问题或项目无法继续。
- `P1`：造成主要功能错误、重大返工或交付失败。
- `P2`：造成局部返工、效率下降或质量风险。
- `P3`：轻微问题、体验问题或优化建议。

### PIT-2026-005：iOS 自动生成 Info.plist 导致中文显示名编码错误

- 日期：2026-06-23
- 项目：project-001-个人雅思单词记忆
- 分类：实现 / 交付
- 严重程度：P1
- 状态：已验证
- 发生阶段：AltStore IPA 最终验包
- 现象与事实：IPA 的 Bundle ID 正确，但 `CFBundleDisplayName` 从包内 plist 解析为乱码，不是用户要求的“单词TUO”。
- 造成的影响：即使 App 可安装，iPhone 主屏幕名称也可能显示错误，影响最终交付验收。
- 根本原因：通过 XcodeGen build setting 自动生成 Info.plist 时，中文显示名在构建链路中存在编码风险。
- 解决过程：改为显式 `Info.plist`，使用 XML 数字实体写入 `&#x5355;&#x8BCD;TUO`，重新触发 GitHub Actions 构建并验包。
- 最终方案：最终 IPA 的 `CFBundleDisplayName` 编码点为 `\u5355\u8bcdTUO`，Bundle ID 为 `com.personal.wordtuo`。
- 预防措施：含中文 App 名的 iOS 包必须解析最终 IPA 内的二进制 plist，不能只看源码或文件名。
- 可复用结论：交付 iOS IPA 时，显示名、Bundle ID 和资源版本都要从最终包内验证。
- 相关文件或证据：`project-001-个人雅思单词记忆/04-实现/ios/Info.plist`、`project-001-个人雅思单词记忆/05-测试与验收/05-v4测试结果.md`
- 后续验证日期：2026-06-23

### PIT-2026-004：把外部搜索入口误当作影视语境功能

- 日期：2026-06-22
- 项目：project-001-个人雅思单词记忆
- 分类：需求 / 实现 / 交付
- 严重程度：P1
- 状态：已验证
- 发生阶段：v2 功能验收
- 现象与事实：页面提供了三个外部搜索入口，但没有保存片段、时间范围或 App 内播放器，用户正确指出“影视语境没做”。
- 造成的影响：实现只覆盖了来源发现，没有完成用户要求的训练闭环。
- 根本原因：把供应商接口受限误解成可以降低核心交互验收标准。
- 解决过程：将功能重构为外部搜索、保存合法公开视频链接、限制5–20秒、懒加载播放器和持久化收藏。
- 最终方案：没有自动搜索接口时，也必须提供真实可操作的替代闭环，并明确哪些环节需要用户参与。
- 预防措施：功能验收按用户动作闭环检查，不能用“有入口”代替“可完成任务”。
- 可复用结论：降级方案可以降低自动化程度，但不能只保留跳转链接而宣称核心功能完成。
- 相关文件或证据：`project-001-个人雅思单词记忆/05-测试与验收/04-v3测试结果.md`
- 后续验证日期：2026-06-22

### PIT-2026-003：PWA 缓存升级混用新 HTML 与旧 JavaScript

- 日期：2026-06-21
- 项目：project-001-个人雅思单词记忆
- 分类：实现 / 测试
- 严重程度：P1
- 状态：已验证
- 发生阶段：v2 发布后的真实浏览器刷新
- 现象与事实：无痕自动测试通过，但长期运行的浏览器刷新后显示新版入口，事件绑定仍执行旧版脚本并报空元素错误。
- 造成的影响：页面视觉升级但核心逻辑没有启动，数据指标显示为0。
- 根本原因：仅修改缓存名称不足以保证历史 Service Worker 不返回同名旧资源。
- 解决过程：读取真实浏览器控制台，确认 `app.js` 版本混用；为 CSS 和 JavaScript URL 增加版本参数，同时升级缓存名。
- 最终方案：发布结构性前端更新时同时版本化资源 URL 与 Cache Storage 名称。
- 预防措施：PWA 发布测试必须覆盖已有 Service Worker 控制的浏览器，不能只测全新上下文。
- 可复用结论：缓存迁移是发布兼容性的一部分；无痕测试无法替代历史缓存升级测试。
- 相关文件或证据：`project-001-个人雅思单词记忆/04-实现/web/index.html`、`sw.js`
- 后续验证日期：2026-06-21

### PIT-2026-001：未锁定产品目标导致错误交付聆听训练 IPA

- 日期：2026-06-21
- 项目：project-001-个人雅思单词记忆
- 分类：需求 / 目录 / 交付
- 严重程度：P1
- 状态：已解决
- 发生阶段：产品实现与 iPhone 部署
- 现象与事实：原始目标是个人雅思单词记忆 App，但工作区主线被改成“聆听训练”，随后构建和部署使用了错误 IPA。
- 造成的影响：完成了错误产品的 AltStore 部署准备，并产生多次改名和重打包返工。
- 根本原因：没有在构建前再次核对产品目标、Bundle ID、显示名和核心数据模型；项目文件散落在 Workspace 根目录。
- 解决过程：回顾对话，停止使用 ListeningTraining IPA，建立独立合规项目目录。
- 最终方案：单词记忆项目使用独立目录、独立 Bundle ID `com.personal.wordtuo` 和强制验包清单。
- 预防措施：每次 IPA 构建前核对产品说明、Bundle ID、显示名、首页文案、数据键和 Web 资源；不同产品不得共用根目录源码。
- 可复用结论：部署成功不能证明产品正确，交付前必须做需求身份校验。
- 相关文件或证据：`project-001-个人雅思单词记忆/00-项目总览/01-项目说明.md`

### PIT-2026-002：Codex 内置运行时未加入 PATH 导致技能安装回退失败

- 日期：2026-06-21
- 项目：Workspace 环境维护
- 分类：工具
- 严重程度：P3
- 状态：已验证
- 发生阶段：从 GitHub 安装 Codex skills
- 现象与事实：技能安装器在直接下载失败后回退到 Git，但当前 PowerShell 的 `PATH` 中没有 Codex 内置 Git；调用 skills.sh CLI 时也因内置 Node 未加入 `PATH` 而失败。
- 造成的影响：`planning-with-files` 首次安装中断，skills.sh 首次查询失败。
- 根本原因：Codex Desktop 提供了隔离的 Git、Node 和 Python 运行时，但不会保证这些可执行文件出现在普通 shell 的全局 `PATH` 中。
- 解决过程：通过 Workspace Dependencies 获取内置运行时绝对路径，在单次命令中临时加入 `PATH` 后重试；并为默认分支为 `master` 的仓库显式传入 `--ref master`。
- 最终方案：安装或查询技能前先定位内置运行时；外部仓库同时核对默认分支，避免将 `main` 作为固定假设。
- 预防措施：涉及技能安装、`npx`/`pnpm` 或 Git 回退时，先加载 Workspace Dependencies，并在当前进程设置 Git 与 Node 路径。
- 可复用结论：桌面端“已提供运行时”不等于“shell 可直接按命令名调用”；自动化脚本应使用已发现的绝对路径或显式设置进程级 `PATH`。
- 相关文件或证据：`C:\Users\妥妥的\.codex\skills\.system\skill-installer\scripts\install-skill-from-github.py`
- 后续验证日期：2026-06-21

### METHOD-2026-003：AltStore IPA 交付优先使用“下载到文件 App 后分享到 AltStore”

- 日期：2026-06-29
- 来源项目：聆听训练
- 适用场景：Windows 本机生成或托管 IPA，需要用户在 iPhone 上通过 AltStore 安装；尤其适用于局域网页面空白、`altstore://install` 跳转不稳定、中文用户名导致 AltServer 临时目录异常等情况。
- 不适用场景：正式 App Store / TestFlight 分发，或用户没有安装并修复可用的 AltStore。
- 前置条件：IPA 已构建完成；iPhone 可通过 Safari、网盘、微信/QQ、iCloud Drive 或局域网 HTTP 下载到“文件”App；AltStore 在 iPhone 上可用。
- 执行步骤：提供直接 IPA 下载链接；用户在 iPhone Safari 下载 `.ipa` 文件；下载完成后从 Safari 下载列表或“文件”App 打开该 IPA；点击分享按钮；选择 AltStore 安装。
- 验证标准：iPhone 能在“文件”App 中看到 IPA；分享菜单能选择 AltStore；AltStore 能开始安装目标 App。
- 可复用结论：以后交付 IPA 时，默认同时提供“直接 IPA 下载”作为主路径，并明确说明“下载到文件 App 后分享到 AltStore”；`altstore://install` 只作为辅助路径，不再作为唯一安装方式。
- 注意事项：“文件”App 只能保存 IPA，不能直接安装；若 AltStore 显示“不再可用”，先用 AltServer 刷新或重装 AltStore；Windows 中文用户目录可能导致 AltServer 临时目录乱码，必要时先将用户级 `TEMP` / `TMP` 改为纯英文路径，例如 `C:\AltStoreTemp`，再重启 AltServer。
